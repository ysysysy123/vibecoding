# Git 基础入门教程

## 一、什么是 Git？

Git 是一个**分布式版本控制系统**，用于追踪文件的变化，方便多人协作开发。

### 核心概念

- **版本控制**：记录文件的每一次修改，随时可以回退到任意版本
- **分布式**：每个人的电脑上都有完整的代码仓库，不依赖中央服务器
- **Git 仓库（Repository）**：存放项目文件和所有版本历史的地方

---

## 二、Git 三大区域（工作区、暂存区、版本库）

这是理解 Git 的关键！Git 管理的文件分为三个区域：

```
┌─────────────────┐    git add     ┌─────────────────┐   git commit    ┌─────────────────┐
│                 │ ──────────────►   │                 │ ──────────────► │                 │
│   工作区        │                │   暂存区         │                │   版本库         │
│  (Working       │                │  (Staging       │                │  (Repository)    │
│   Directory)    │                │   Area)         │                │                 │
│                 │ ◄──────────────   │                 │ ◄─────────────  │                 │
└─────────────────┘   git checkout └─────────────────┘  git reset    └─────────────────┘
```

### 1. 工作区（Working Directory）
- 就是你在电脑里能看到的目录，你直接编辑修改的文件都在这里
- 比如你现在看到的 `/workspace` 文件夹就是工作区

### 2. 暂存区（Staging Area / Index）
- 临时存放你准备提交的修改
- 用 `git add` 命令把工作区的修改放到暂存区
- 相当于"确认哪些修改要提交"

### 3. 版本库（Repository）
- 存放所有已经提交的版本历史
- 用 `git commit` 把暂存区的内容提交到版本库
- 提交后就生成了一个永久的版本记录

## 三、基础命令速查

| 命令 | 作用 |
|------|------|
| `git init` | 初始化一个新的 Git 仓库 |
| `git status` | 查看当前状态（哪些文件修改了、在哪个区域） |
| `git add <文件名>` | 把文件添加到暂存区 |
| `git add .` | 把所有修改的文件添加到暂存区 |
| `git commit -m "提交说明"` | 把暂存区的内容提交到版本库 |
| `git log` | 查看提交历史记录 |
| `git diff` | 查看工作区和暂存区的差异 |
| `git diff --staged` | 查看暂存区和版本库的差异 |
| `git checkout -- <文件名>` | 丢弃工作区的修改 |
| `git reset HEAD <文件名>` | 把暂存区的修改撤回到工作区 |

## 四、实战操作示例

### 第一步：查看当前状态

```bash
git status
```

输出会告诉你：
- 当前在哪个分支
- 哪些文件修改了但还没 add（在工作区）
- 哪些文件已经 add 了但还没 commit（在暂存区）

### 第二步：修改文件后添加到暂存区

```bash
# 添加单个文件
git add README.md

# 添加所有文件
git add .
```

### 第三步：提交到版本库

```bash
git commit -m "修改了 README 文件，添加了项目说明"
```

> 💡 **小技巧：提交说明要写清楚这次改了什么，方便以后回看

### 第四步：查看提交历史

```bash
git log

# 简洁版
git log --oneline
```

## 五、撤销操作

### 场景1：修改了文件但还没 add，想撤销修改

```bash
git checkout -- 文件名
# 或者（Git 新版本推荐）
git restore 文件名
```

### 场景2：已经 add 了但还没 commit，想撤回

```bash
git reset HEAD 文件名
# 或者
git restore --staged 文件名
```

### 场景3：已经 commit 了，想回退到上一个版本

```bash
git reset --hard HEAD^
```

- `HEAD` 表示当前版本
- `HEAD^` 表示上一个版本
- `HEAD~10` 表示往上10个版本
- 也可以用提交ID（commit hash）指定具体版本

## 七、`.gitignore` 文件

有些文件不需要 Git 管理，可以创建 `.gitignore` 文件来忽略它们：

```
node_modules/
*.log
.DS_Store
dist/
```

这样 Git 就会自动忽略这些文件/目录。
